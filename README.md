# منصة الخطة التشغيلية المدرسية — الإصدار المُحدّث

## الملفات
- `index.html` — الصفحة الرئيسية والواجهة الكاملة
- `style.css` — التصميم والهوية البصرية (متجاوب للجوال)
- `script.js` — منطق التطبيق + اتصال Supabase
- `config.js` — إعدادات Supabase (URL + anon key فقط)

## خطوات التشغيل
1. أنشئ مشروعاً في Supabase.
2. من **SQL Editor** نفّذ كود SQL الموجود في أعلى ملف `script.js` (داخل التعليق).
   - إن كان جدول `teacher_followups` موجوداً مسبقاً، ستضيف أوامر `ALTER TABLE`
     عمودي `drive_link` و `created_by` تلقائياً دون فقد البيانات.
3. افتح `config.js` وضع:
   - `SUPABASE_URL` = رابط المشروع
   - `SUPABASE_ANON` = مفتاح anon العام (Project Settings → API)
   - ⚠️ لا تضع `service_role` إطلاقاً في الواجهة الأمامية.
4. ارفع الملفات الأربعة على أي استضافة ثابتة (Netlify / Vercel / GitHub Pages…).

## ملخص التعديلات المنفذة
1. هوية بصرية محدّثة (ألوان فيروزي/زمردي + ذهبي) واسم محترف للمنصة.
2. الإعدادات (اسم المدرسة/القائدة/العام) تُحفظ في Supabase + LocalStorage وتبقى بعد التحديث.
3. اسم القائدة يظهر في الصفحة الرئيسية ويتحدّث فور حفظ الإعدادات.
4. الأدوار: مدير / وكيل / معلمة فقط في الواجهة.
5. مؤشرات الإنجاز تُحتسب تلقائياً، وكذلك من الشواهد المرتبطة عند غياب المؤشرات.
6. أُزيلت إضافة الشواهد من بطاقة البرنامج؛ تُضاف فقط من «التقارير والشواهد».
7. شواهد المعلمة تُربط بالبرنامج وتظهر في التقارير وداخل تفاصيل البرنامج.
8. زر التفاصيل يعرض: البيانات الكاملة، الشواهد، روابط درايف، حالة الإنجاز، واسم المعلمة الرافعة.
9. تعديل البرنامج لا يحذف الشواهد المرتبطة (الربط عبر `program_id`).
10. متابعة المعلمات: المعلمة تُرسل رابط Drive فقط (دون عرض/تعديل/حذف)؛ المدير والوكيل يديرونها بالكامل.
11. جميع الإضافات/التعديلات/الحذف محفوظة في Supabase وتبقى بعد التحديث.
12. تحسين التجاوب على الجوال (قوائم، جداول قابلة للتمرير، أزرار ممتدة).
13. anon key فقط في الواجهة — بلا service_role.
